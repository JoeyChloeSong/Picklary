# IP 기반 언어 자동 설정 업그레이드

이번 버전은 사용자가 사이트 루트(`/`)로 접속할 때 자동으로 언어 페이지를 선택합니다.

## 기본 라우팅 규칙

- 한국(`KR`) 접속: `/ko/`
- 중남미 국가 접속: `/es/`
- 그 외 국가: `/en/`

중남미 매핑 국가는 `data/site.config.js`의 `languageDetection.countryLocaleMap.es`에서 수정할 수 있습니다.

## 작동 방식

정적 사이트는 브라우저 JavaScript만으로 방문자의 IP를 직접 읽을 수 없습니다. 그래서 루트 페이지는 아래 순서로 대략적인 국가 코드를 확인합니다.

1. 같은 도메인의 Cloudflare trace(`/cdn-cgi/trace`)가 있으면 그 값을 사용
2. 없으면 공개 GeoIP JSON endpoint(`https://ipapi.co/json/`) 사용
3. 둘 다 실패하면 브라우저 언어를 참고
4. 그래도 판단이 어려우면 영어(`/en/`)로 이동

사용자가 상단 언어 선택 메뉴에서 직접 언어를 고르면 `localStorage`의 `picklelevel.lang` 값이 저장되고, 다음 루트 접속부터는 직접 선택한 언어가 우선됩니다.

## AdSense/개인정보 관점 메모

공개 페이지에는 자동화 에이전트 문구가 들어가지 않으며, 언어 자동 설정은 사용자 편의 기능으로만 동작합니다. 개인정보처리방침과 쿠키 정책에는 국가 코드 기반 언어 안내와 브라우저 저장소 사용 설명을 추가했습니다.

IP 기반 국가는 VPN, 프록시, 모바일 통신망, 회사망 때문에 실제 위치와 다를 수 있습니다. 사용자가 언제든 상단 언어 메뉴에서 한국어/영어/스페인어를 직접 선택할 수 있게 유지하는 것이 좋습니다.

## 로컬 테스트

로컬 실행은 기존과 같습니다.

```powershell
00_RUN_SITE_WINDOWS.bat
```

이제 브라우저는 `/ko/`가 아니라 루트 주소를 먼저 엽니다.

```text
http://127.0.0.1:8787/
```

직접 확인할 수 있는 언어별 주소는 다음입니다.

```text
http://127.0.0.1:8787/ko/
http://127.0.0.1:8787/es/
http://127.0.0.1:8787/en/
```

## 운영 전 확인

스페인어 사용자에게 `/es/`를 제공하기 위해 `data/site.config.js`의 locales에 `es`를 다시 포함했습니다. 다만 전체 본문까지 완전한 스페인어 번역을 갖추려면 향후 주요 가이드, 패들, 선수 페이지의 스페인어 번역을 단계적으로 보강하는 것을 추천합니다.
