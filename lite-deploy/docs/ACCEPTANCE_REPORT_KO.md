# Picklary Lite v0.7.5 완전 통합 검증 보고서

## 요청 반영

1. DualCam에서 두 영상을 불러온 뒤 재생 버튼과 타임라인이 영상 표면에 가려지는 문제를 구조적으로 수정했습니다.
2. A/B 영상 명칭을 출력 위치 기준으로 변경했습니다.
3. 한글판과 영문판을 동일한 처리 구조로 제작했습니다.
4. 두 Windows ZIP을 포함하는 Netlify 배포 패키지를 별도로 구성했습니다.

## 수정 원리

기존 DualCam 미리보기는 Windows 네이티브 자식 표면을 만드는 `QVideoWidget`을 사용했습니다. 통합 창, 스크롤, DPI 배율 및 두 개의 동시 영상 표면이 결합되면 네이티브 표면의 Z-order가 일반 Qt 조작부보다 앞에 남아 재생 버튼과 타임라인을 덮을 수 있습니다.

v0.7.5는 DualCam도 `QVideoSink`를 사용합니다. 수신한 프레임을 일반 `QWidget` 캔버스에 직접 그려 영상, 버튼 및 타임라인이 동일한 레이아웃과 Z-order 안에서 처리됩니다. 페이지 전체 스크롤도 추가했습니다.

## 검증 한계

현재 제작 환경에는 PySide6 Windows GUI와 NVIDIA GPU가 설치되어 있지 않아 실제 Windows 영상 표면을 눈으로 확인하는 실행 검증은 수행하지 못했습니다. 따라서 새 폴더에 압축을 푼 뒤 사용자 Windows PC에서 두 영상을 불러와 재생 버튼과 타임라인이 계속 보이는지 최종 확인해야 합니다.
